1. 만들기 4가지 유형

Prototype클릭되는 화면 시안 — 와이어프레임/고해상도 선택
Slide deck발표자료 — 발표자 노트(speaker notes) 옵션
Template반복해 쓰는 서식·틀 만들기
Other그 외 — 카드뉴스·포스터·문서 등 자유 제작
  • Prototype(프로토타입) — 버튼이 눌리고 화면이 넘어가는 시안. 처음엔 High fidelity(완성도 높은 화면)를 고르면 됩니다.
  • Slide deck(슬라이드 덱) — 발표용 슬라이드. 발표 멘트가 필요하면 Use speaker notes를 켭니다.
  • Template(템플릿) — 매번 비슷하게 쓰는 안내문·카드 틀을 만들어 두고 재사용.
  • Other(기타) — 위 분류에 없는 자유로운 결과물(카드뉴스·포스터·한 장 문서 등).

처음엔 이렇게

일반 수강생은 대부분 Prototype → High fidelity 또는 Slide deck으로 시작하면 충분합니다.

일반 수강생은 직접 만들 필요 없이 이미 있는 시스템을 골라 Done을 누르면 됩니다. (예: reveal.js Design System · Default) 직접 만들려면 ＋ Set up a design system을 누르세요 — 아래 3단계로 이어집니다.

3. (심화) Set up a design system — 내 시스템 직접 만들기

회사·동호회만의 색과 글꼴이 있다면, 위 목록의 ＋ Set up a design system을 눌러 나만의 시스템을 직접 만들 수 있어요. 원하는 사람만 하는 선택 과정이라 맨 마지막에 다룹니다.

1회사 소개(Company name and blurb)를 한 줄 적습니다 — 무엇을 하는 곳인지, 어떤 분위기인지.
2자료 첨부(모두 선택) — 깃허브 코드·내 컴퓨터 폴더·.fig 파일, 그리고 ‘Add fonts, logos and assets’에 DESIGN.md와 폰트 파일을 끌어다 놓습니다.
3클로드가 만든 초안을 검토하고, 좋으면 Published(게시)에 체크 — 이후 새 프로젝트의 기본 스타일이 됩니다.

md 파일·폰트 넣는 핵심

① DESIGN.md — 색·글꼴·분위기를 적은 문서. ‘Add fonts, logos and assets’ 칸에 끌어다 놓으면 시스템이 한 번에 차려집니다. (견본은 강의 ‘참고자료’에서)
② 폰트 — 회사 전용·유료 글꼴이면 같은 칸에 .woff2 / .ttf 파일을 올려요. 안 올리면 초안 검토에서 ‘Missing brand fonts’로 표시됩니다.
